1.1 核心稽核思維：三大防線第一防線：無缝 UDI 唯一性追溯（Traceability Guard）任何植入性醫療器材必須具備全球唯一的唯一裝置識別碼（UDI）。系統內核透過 purchase_records 表格中的 udi_di（產品識別碼）與 serialNo（生產識別碼 / 序號，即 UDI-PI）建立一對一的絕對關係。稽核官可藉此杜絕平行輸入（水貨）、翻新件與拼裝件流入臨床體系。第二防線：動態無菌失效與庫存降級稽核（Sterility & Expiry Audit）高價值植入物具備嚴格的無菌屏障壽命。系統利用先進的先進先出（FIFO）演算法與動態衰減公式，將剩餘無菌天數（savedDays）低於 180 天的裝置自動標記為「過期警戒黑洞」，防止過期醫療器材危害患者生命。第三防線：主動式跨節點再平衡（Active Node Rebalancing）當特定醫學中心面臨緊急手術導致安全庫存告急，而鄰近機構存有即期呆料時，系統將結合地理資訊系統（GIS）與拓撲網絡，主動計算摩擦係數並生成調度指令，阻斷區域性缺貨風險。2. 虛擬 SQLite 數據庫 schema 解析與稽核查詢DHA-Hub 內部運行一個高性能的虛擬 SQLite 關係型引擎，所有的採購、分銷、合規覆蓋記錄皆受到資料庫完整性約束（Integrity Constraints）。稽核官在進行深度數據探勘時，可直接呼叫內建的終端或透過 AI 指揮官執行以下核心 Table 的稽核查詢：2.1 採購與進貨核心明細表 (purchase_records)本表儲存所有經由合法報關、檢驗通關進入市場的醫療器材原始源頭。SQLCREATE TABLE purchase_records (
