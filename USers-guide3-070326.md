2.1 雙層模型網關路由與隱私安全運算層 (Local App)： 預設調用 gemini-3.1-flash-lite。該模型負責讀取資料集的 Schema、Metadata 與 3 行樣本，在本地自主編譯出一萬行級別的 Pandas 矩陣清洗、NetworkX 拓撲圖論計算與 Scipy 機率模型程式碼。隨後在本地安全沙箱環境執行。此層確保敏感的病患隱私、醫院採購價格等原始數據絕不離開本地設備。推理層 (Cloud High-End App)： 預設調用 gemini-3.5-flash（可一鍵切換至 gemini-3.1-pro-preview 等高階模型）。沙箱執行完畢後，僅將計算完畢的高壓縮統計指標、黑洞序號清單（體積通常 < 5KB）回傳至雲端。推理層以此精簡特徵為藍本，注入歷史狀態快照（State-DB），撰寫出具備法律效力與政策建議深度的千字級官方稽查策略報告。3. 資料來源物理結構與法規語意解析 (Data Schema & Semantics)稽查官每月導入口、產、銷資料時，系統會自動將三個核心數據集進行異質對齊。以下為系統內存的標準數據結構與語意規範：3.1 採購與進貨數據集 (Purchase Dataset)記錄各醫療機構、一級通路商向境外製造商或台灣總代理進貨之明細。序號 (Seq_No)： Integer | 唯一流水號。申報業者 (Declarer_ID)： String | 進貨法人主體統一編碼（如 A00013 代表台大醫院、A00002 代表台北榮總）。在圖論中，此為網路的終端匯聚節點 (Sink Nodes)。收貨日期 (Receive_Date)： Date | 格式標準化為 YYYYMMDD。供應商 (Supplier_ID)： String | 上游製造商或進口總代理商（如 C00306 代表美敦力總代理）。在圖論中，此為網路的源點 (Source Nodes)。許可證號 (License_No)： String | 衛福部核發之醫療器材許可證字號（如「衛部醫器輸字第030747號」）。全球品項識別碼 (UDI_DI)： String | GTIN-14 全球統一編碼。產品序號 (Serial_No)： String (Unique) | 核心碰撞主鍵。每一顆植入物的全球唯一身分證字號（如 RNJ146480G2001）。有效期間 (Exp_Date)： Date | 產品貨架壽命截止日。保存期限 (Shelf_Life_Days)： Integer | 系統自動根據 $Exp\_Date - Current\_Date$ 計算之剩餘天數。退貨資訊 (Return_Status)： Integer | 0 代表正常進貨，1 代表退貨。3.2 通路與分銷數據集 (Distribution Dataset)記錄中游經銷商將產品交貨至下游終端對象（醫療機構）的銷貨明細。對齊難點與消解規則： 通路表中的產品編碼常被誤命名為 UDID（採購表為 UDI_DI）；型號欄位常夾雜非結構化文字雜訊（如「# 美敦力亞士爾磁振造影植入式心臟節律器-單腔 W2SR01」）。Local App 會自動生成正則表達式（Regex），將其清洗並提取出標準型號 W2SR01。3.3 地理網絡站點數據集 (Geolocation Stations)提供各實體機構節點的空間地理資訊（JSON 格式）。包含 entity_id、official_name、entity_type（如 Distributor、Hospital_Group）、postal_code 及 WGS-84 經緯度座標（latitude / longitude）。4. 稽查官核心武器：三大 AI 前瞻功能與法規演算法DHA-Hub 為稽查官配置了三大核心 AI 監管引擎。以下詳述其背後的法規邏輯、數學演算法與 API 規格：4.1 智慧型醫材法規回收與風險追溯追蹤引擎法規背景： 當原廠發布主動回收指令時，稽查官必須在黃金 24 小時內定位全台風險醫材。演算法原理： 系統調用 gemini-3.1-flash-lite 解析非結構化法規回收公告，將其轉化為結構化篩選字典。隨後在本地 NetworkX 構建的有向有權圖中，執行 全路徑生命週期碰撞遍歷（Lifecycle Sequence Matching）。若發現某唯一序號已由經銷商出貨給醫院，但該醫院系統中卻無該序號的「採購收貨申報」，系統立即將該路徑標記為「失控高危節點（追溯黑洞）」。核心 API 規格 (POST /api/ai/recall-trace)：JSON{

回傳結果包含提取的結構化規則、受影響總數、在庫/在途/已植入數量，以及高危失控品項的完整流轉歷史軌跡。4.2 多維前置時間預測與多目標庫存重平衡引擎監管背景： 預防醫療機構惡意囤貨（Hoarding）導致區域性醫療資源斷鏈，或因庫存呆滯導致醫材過期。演算法原理： 系統計算申報系統寫入時間戳與實體收貨日期的延遲差，利用指數平滑法（Exponential Smoothing）預測各節點的「真實前置時間（True Lead-Time）」。隨後結合地理座標（$Lat_i, Lon_i$），求解以下 多目標重平衡優化模型 (Multi-Objective Optimization)：$$\min \quad w_1 \sum_{i,j} X_{ij} \cdot D_{ij} + w_2 \sum_{j} \max(0, S_j - I_j) + w_3 \cdot P_{\text{compliance}}$$其中 $X_{ij}$ 為調撥量，$D_{ij}$ 為使用哈維辛公式（Haversine Formula）計算之大圓航線公里數，$S_j$ 為安全庫存警戒線，$P_{\text{compliance}}$ 為合規懲罰權重。4.3 醫材申報合規交叉審計與自動處方建議系統法規背景： 依據《醫療器材管理法》，高風險醫材流向申報必須在出/收貨完成後 15 日內完成。演算法原理（時序因果律檢驗 Temporal Causality Check）： 對於同一個唯一序號 $\lambda$，檢索其在通路表（出貨）的時間戳 $T_{\text{dist}}$ 與在採購表（入庫）的時間戳 $T_{\text{pur}}$。驗證法規約束條件：$$0 \le T_{\text{pur}} - T_{\text{dist}} \le 15 \, \text{days}$$若 $T_{\text{pur}} < T_{\text{dist}}$，系統判定為「時序逆轉（涉嫌未買先賣、非法串貨或虛偽申報）」；若時間差 $> 15$，則判定為「申報逾期違規」，系統自動援引法條並計算預估罰鍰。5. 互動式監管儀表板：五大 Wow 級互動圖表操作指南DHA-Hub 儀表板採用高對比科技微光風格（High-Contrast Tech-Glow
