2.1 雙層 Agent 運作工作流第一層：本地核心應用 (Local App Agent - gemini-3.1-flash-lite)當稽查官導入年度或月度醫材申報資料集時，Local App Agent 會首先讀取採購（Purchase）與通路（Distribution）的原始數據 Schema。接著，它會動態編譯出高容錯性的數據清洗與圖論計算 Python 腳本（clean_and_align.py），並投射到本地沙箱執行。此層任務不處理實體大數據的文本發送，而是將數百萬筆交易明細壓縮為少於 5KB 的分析特徵矩陣（Aggregated Features JSON）。第二層：高階雲端推理應用 (SUPER LLM Agent - gemini-3.5-flash)高階推理 Agent 讀取由 Local App 傳遞的壓縮 JSON 矩陣，並從本地 State-DB 中動態拉取歷史月份的趨勢特徵快照。透過滑動視窗（Sliding Window）機制，進行跨時序的深度演繹推理，最終撰寫出符合 FDA 法規術語與合規要件的「月度通路與採購策略稽查報告」。2.2 數據安全與 Docker 輕量化隔離沙箱 (gVisor 運行時)為了防止 LLM 因程式碼幻覺或遭到惡意提示詞注入（Prompt Injection）而產生具破壞性的系統指令，DHA-Hub 配置了工業級的安全沙箱隔離技術：核心核心隔離： 本地生成的 Python 腳本被強制限制在 Google 開源的 gVisor 容器運行時中執行。gVisor 藉由攔截並在用戶空間重新實現所有系統調用，提供了強大的核心級隔離。網路中斷（Air-gapping）： 沙箱容器在啟動時其網路配置設為 network: none，完全切斷與外部網際網路的實體 Socket 聯絡，徹底防範醫療 UDI 數據的外洩。唯讀掛載與儲存限制： 原始數據目錄是以「唯讀（Read-Only）」權限掛載到沙箱中。沙箱內部僅允許將運算後的特徵 JSON 檔案寫入容量限制為 1MB 的 /tmp/output.json 目錄，其餘任何寫入行為皆會觸發 Permission Denied 異常並立即銷毀容器。3. 數據源架構與欄位語意解析 (Data Schema & Semantics)DHA-Hub 的分析準確性建立在對三項核心資料集進行精準語意對齊的基礎上。稽查官在審查廠商申報資料時，應密切注意以下核心欄位的結構與關聯性：+---------------------------------------------------------------------------------+

隨後，系統將其與內置的衛生福利部食品藥物管理署 (TFDA) 醫療器材法規分類碼（如 E.3610 植入式心律器之脈搏產生器）進行語意碰撞與自動修正（Self-Healing），並在臨時內存中覆寫標準化 UDI。監管與視覺呈現： 系統會在 CLI 控制台即時輸出「資料清洗對齊 Live 日誌」，詳細展示 AI 是如何過濾噪音、還原標準法規代碼的過程。稽查官不需手動編寫清洗腳本，即可獲得完全對齊的法規審計基礎。4.3 AI Feature 3：主動式過期預警與庫存調配智慧經紀人技術運作機制： 高風險植入性醫材單價極高（常達數十萬新台幣），且其包裝與生物塗層具有嚴格的有效期限。本功能透過 逆向推估演算法 (Backward Imputation) 結合 State-DB 歷史消耗率進行預測。系統主動掃描在下游院所已進貨但無對應出貨紀錄的唯一產品序號，依據其「有效期間」計算剩餘天數。若某醫學中心（如 A00013）有 5 個剩餘庫存，其剩餘有效期限為 6 個月，但該院在 State-DB 中的歷史月平均消耗速度僅 0.2 個，系統將利用以下公式計算其過期機率：$$\text{Expiry Probability} = 1 - P(X \ge K)$$當預估過期機率超過預設閾值（如 $70\%$）時，高階推理 Agent 會自動尋找同區域內消耗速度極快、庫存告急的其他院所節點（如 A00002），並自動撰寫一份符合企業營運與法規利益的「調配策略合約提案」。監管與視覺呈現： 系統將在「產品有效期限與失效風險 Temporal 時序矩陣熱圖」中，將該高危庫存網格以「脈動發光的霓虹粉紅色」高亮顯示，並提供一鍵通知相關院所資材科與代理商的 Webhook 按鈕，將報廢損失降至最低。5. 極致前端視覺效果與 UI/UX 設計系統 (The Wow UI/UX)DHA-Hub 專為高難度監管場景設計了具備數位孿生感（Digital Twin）與控制台美學的「高端深色控制面板 (Default Dark Theme)」與「明亮無邊界設計 (Light Theme)」，並全面支援繁體中文與英文雙語切換。5.1 關鍵 UI/UX 組件與互動指標+---------------------------------------------------------------------------------+
